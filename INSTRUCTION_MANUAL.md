# CHUWI 13 스마트 현황판 사용 설명서

이 폴더는 `dashboard` 하나만 CHUWI 노트북에 넣으면 바로 실행할 수 있도록 정리된 독립형 현황판입니다.

이 설명서는 초보자도 그대로 따라할 수 있게 아주 천천히 적었습니다.

---

## 1. 가장 중요한 핵심

이제 `dashboard` 폴더만 있으면 됩니다.

이 폴더 안에 현황판 실행, 화면 표시, 날씨, 학교 시간표, 학원 일정 파싱, Firebase 할일 연결에 필요한 파일이 모두 들어 있습니다.

즉, 다른 프로젝트 폴더를 지워도 `dashboard` 폴더만 남겨두면 현황판을 실행할 수 있게 구성한 것입니다.

---

## 2. 이 현황판이 지금 보여주는 것

현재 이 현황판은 아래 정보를 한 화면에 모아서 보여줍니다.

- 오늘 / 내일 날씨 요약
- 현재 시간 / 날짜
- Firebase 실시간 할일
- 숙제
- 약속 / 모임
- 오늘 학교 시간표
- 오늘 학원 일정
- StudySet 01 영어 단어 카드
- StudySet 02 국어 단어 카드
- 전체 / 오늘 마감 / 약속 / 지연 개수

중요한 방향:

- 페이지 전체 스크롤 없이 한 화면 중심
- 큰 장식보다 정보량 우선
- 빈 데이터는 큰 박스 대신 짧고 compact하게 표시
- 넘치는 데이터는 각 패널 안에서만 스크롤

## 3. 폴더 안 파일 설명

- `index.html`
  현황판 화면입니다.

- `dashboard.css`
  다크 테마, 카드 스타일, dense 레이아웃을 담당합니다.

- `dashboard.js`
  시계, 날씨, Firebase 할일, 학교 시간표, 학원 일정 렌더링을 담당합니다.

- `dashboard-config.js`
  Firebase 정보, 날씨 지역, 학교 시간표 데이터가 들어 있습니다.

- `cram-parser.js`
  `cram/cram.md`를 읽어서 학원 일정 데이터로 바꾸는 파서입니다.

- `cram/cram.md`
  학원 일정 원본 파일입니다. 앞으로 학원 일정은 이 파일만 수정하면 됩니다.

- `memory-parser.js`
  `memory/*.md` 파일을 읽어서 암기 카드 데이터로 바꾸는 파서입니다.

- `memory/english.md`, `memory/korean.md`, `memory/science.md`, `memory/social.md`, `memory/idiom.md`, `memory/hanmun.md`
  지금 바로 테스트할 수 있도록 넣어둔 예시 암기 카드 파일들입니다.

- `local_secrets.env`
  실제 Gemini API 키를 서버만 읽도록 저장하는 로컬 전용 설정 파일입니다. 이 파일은 GitHub에 올리지 않는 것이 안전합니다.

- `local_secrets.env.example`
  GitHub에 올려도 되는 예시 설정 파일입니다. 실제 키는 넣지 않고 형식만 보여줍니다.

- `GEMINI_STUDYSET_SETUP.md`
  Gemini가 매일 의미 있게 StudySet 2개를 고르도록 설정하는 설명서입니다.

- `serve_dashboard.py`
  작은 로컬 서버입니다.

- `start_dashboard.bat`
  윈도우에서 가장 쉽게 실행하는 파일입니다.

- `start_dashboard.command`
  맥에서 실행할 때 쓰는 파일입니다.

- `manifest.webmanifest`
  웹앱처럼 보이게 돕는 설정 파일입니다.

- `sw.js`
  캐시와 오프라인 보조를 위한 서비스 워커입니다.

- `icon-dashboard.svg`
  현황판 아이콘입니다.

- `INSTRUCTION_MANUAL.md`
  지금 보고 있는 설명서입니다.

- `MACBOOK_AIR_INSTRUCTION_MANUAL.md`
  지금 이 맥북에어에서 바로 실행할 때 쓰는 붙여넣기용 설명서입니다.

- `GITHUB_UPDATE_AND_DOWNLOAD_GUIDE.md`
  맥북에서 GitHub로 올리는 방법, 현황판의 `업데이트` 버튼 동작, 윈도우에서 GitHub 저장소를 내려받는 방법을 정리한 설명서입니다.

- `LAYOUT_EXTENSION_PROMPT.md`
  나중에 기능을 추가할 때 현재 레이아웃 방향을 유지하도록 요청하는 프롬프트입니다.

## 4. 가장 쉬운 실행 방법

CHUWI가 윈도우 노트북이라면 아래 순서를 추천합니다.

1. `dashboard` 폴더를 CHUWI 노트북으로 복사합니다.
2. 복사한 `dashboard` 폴더를 엽니다.
3. `start_dashboard.bat`를 더블클릭합니다.
4. 잠시 기다리면 현황판이 열립니다.

보통 정상 동작하면 브라우저에서 `http://127.0.0.1:8426/index.html` 주소로 열립니다.

이 방법이 좋은 이유:

- `cram/cram.md` 같은 로컬 파일을 안정적으로 읽습니다.
- 브라우저에서 웹앱처럼 쓰기 더 좋습니다.
- 그냥 `index.html`만 여는 것보다 더 안정적입니다.

## 5. `start_dashboard.bat`가 안 될 때

이 실행 파일은 내부적으로 Python으로 작은 서버를 켭니다.

그래서 Python이 설치되어 있어야 가장 잘 작동합니다.

### 먼저 해볼 것

1. `start_dashboard.bat`를 다시 한 번 더블클릭합니다.
2. 그래도 안 열리면 Python 설치 여부를 확인합니다.

### Python이 없다면

1. Python 3를 설치합니다.
2. 설치 화면에서 `Add Python to PATH`를 체크합니다.
3. 설치 후 다시 `start_dashboard.bat`를 실행합니다.

### 급할 때 임시 방법

1. `index.html`을 더블클릭합니다.
2. 브라우저가 열리면 `F11`을 눌러 전체화면으로 봅니다.

이 방법은 임시 확인용으로는 괜찮지만,
가능하면 `start_dashboard.bat` 실행 방식을 추천합니다.

윈도우에서 아래를 확인해 주세요.

1. `설정`
2. `시스템`
3. `전원 및 배터리`
4. 전원 연결 상태에서
5. 절전 안 함
6. 화면 꺼짐 시간도 길게 설정

상황판은 계속 켜둘 목적이므로 전원 연결을 추천합니다.

## 7. 부팅하면 자동으로 켜지게 하는 방법

1. `Win + R`을 누릅니다.
2. `shell:startup` 입력 후 엔터를 누릅니다.
3. 시작프로그램 폴더가 열리면 `start_dashboard.bat`의 바로가기를 넣습니다.

이렇게 하면 로그인 후 자동 실행되기 쉬워집니다.

## 8. 데이터는 어디서 오나요

이 현황판은 크게 5가지 데이터를 읽습니다.

### 1. 할일

Firebase에서 실시간으로 읽습니다.

중요:
주기적으로 전체를 계속 다시 읽는 방식이 아니라,
변경이 생기면 자동 반영되는 실시간 방식입니다.

### 2. 날씨

Open-Meteo 예보를 읽어서 오늘과 내일 날씨를 보여줍니다.

기본 지역은 `서울`입니다.

### 3. 학교 시간표

`dashboard-config.js` 안에 들어 있습니다.

오늘 요일에 맞는 항목만 화면에 표시됩니다.

### 4. 학원 일정

`cram/cram.md` 파일을 읽습니다.

앞으로 학원 일정은 코드 수정이 아니라
`cram/cram.md` 파일만 수정하면 됩니다.

### 5. 암기 카드

이제 `memory` 폴더 안의 `*.md` 파일 전체가 후보가 됩니다.

이 카드는 단어와 뜻이 번갈아 나오고, 한 번에 2개씩 표시됩니다.

교체 간격은 코드가 아니라 각 MD 파일 안의 `@interval:` 값으로 정합니다.

Gemini API 키를 넣으면 오늘 일정과 최근 이력을 참고해서
`StudySet 01`, `StudySet 02`를 의미 있게 2개 고릅니다.

## 9. 학원 일정 수정 방법

가장 자주 수정하게 될 파일은 이것입니다.

- `cram/cram.md`

이 파일은 반드시 아래 형식을 따라야 합니다.

```md
# cram schedule

## 월요일
- 16:30-18:00 | 수학
- 18:10-19:40 | 영어

## 화요일
- 17:00-18:30 | 과학

## 수요일
- 16:30-18:00 | 국어

## 목요일
- 18:00-20:00 | 수학 심화

## 금요일
- 17:30-19:00 | 영어
- 19:10-20:40 | 과학

## 토요일

## 일요일
```

### 꼭 지켜야 하는 규칙

- 요일 제목은 반드시 `## 월요일` 형식
- 일정 줄은 반드시 `- HH:MM-HH:MM | 수업명` 형식
- 공백 줄은 있어도 됨
- 형식이 틀린 줄은 앱이 멈추지 않고 무시됨
- 형식이 틀린 줄은 브라우저 콘솔에 경고가 남음

### 예시

목요일에 영단어 수업을 하나 더 추가하고 싶다면:

```md
## 목요일
- 18:00-20:00 | 수학 심화
- 20:10-21:00 | 영단어
```

저장 후 화면을 새로고침하면 반영됩니다.

## 10. 암기 카드 수정 방법

암기 카드는 `memory` 폴더 안의 모든 `*.md` 파일이 후보입니다.

지금은 예시로 아래 6개가 들어 있습니다.

- `memory/english.md`
- `memory/korean.md`
- `memory/science.md`
- `memory/social.md`
- `memory/idiom.md`
- `memory/hanmun.md`

특정 파일을 수동으로 꼭 띄우고 싶으면
`memory/studyset-force.txt` 파일에 파일명을 한 줄씩 적으면 됩니다.

예:

```txt
science.md
english.md
```

규칙:

- 파일이 비어 있으면 자동 선택됩니다.
- 파일명이 1개면 그 파일이 우선 선택됩니다.
- 파일명이 2개면 두 파일이 바로 선택됩니다.
- 파일명이 3개 이상이어도 에러 없이 위에서부터 2개만 사용합니다.
- 빈 줄과 `#`으로 시작하는 줄은 무시합니다.

형식은 반드시 아래처럼 맞춰 주세요.

```md
# memory deck
@subject: 영어
@interval: 3

| 단어 | 의미 | 도해 |
| --- | --- | --- |
| achieve | 이루다 | |
| compare | 비교하다 | |
| improve | 향상시키다 | images/english/improve.webp |
```

규칙:

- `@subject:`는 화면에 표시할 과목 이름입니다.
- `@category:`는 카드 종류입니다.
- `@tags:`는 관련 과목/분류 태그입니다.
- `@priority:`는 기본 우선순위입니다.
- `@interval:`는 몇 초마다 단어/뜻 화면이 바뀌는지 정합니다.
- 표 헤더는 반드시 `단어 | 의미 | 도해` 순서입니다.
- `도해` 칸은 비워도 됩니다.
- `도해` 칸에는 `images/english/improve.webp` 같은 상대경로나 이미지 URL을 넣을 수 있습니다.
- `도해` 칸에는 `<img src="./english.assets/image-001.png" ... />` 같은 HTML 이미지 태그도 그대로 넣을 수 있습니다.
- 즉, MD 파일 안에 들어 있는 상대경로를 현황판이 그대로 읽고 화면 크기만 맞춰 보여줍니다.
- 그림이 없거나 경로가 잘못돼도 화면에는 텍스트만 보입니다.
- 형식이 잘못된 줄은 앱이 멈추지 않고 무시합니다.

예를 들어 5초마다 바꾸고 싶다면 이렇게 수정하면 됩니다.

```md
@interval: 5
```

나중에 과학, 사회도 같은 형식의 MD 파일을 만들면 같은 방식으로 확장할 수 있습니다.

Gemini 설정은 `GEMINI_STUDYSET_SETUP.md`를 보면 됩니다.
도해 이미지 생성 비율과 프롬프트는 `DOHAE_PROMPT_GUIDE.md`를 보면 됩니다.

## 11. 학교 시간표 수정 방법

학교 시간표는 `dashboard-config.js` 안에 있습니다.

예를 들어 월요일 데이터는 이런 식입니다.

```javascript
monday: [
    { day: "monday", startTime: "09:10", title: "과학" },
    { day: "monday", startTime: "10:05", title: "사회" }
]
```

여기서 `startTime`과 `title`을 바꾸면 됩니다.

---

## 12. 날씨 지역 바꾸는 방법

`dashboard-config.js` 파일 안의 아래 부분을 수정하면 됩니다.

```javascript
weather: {
    locationName: "서울",
    latitude: 37.5665,
    longitude: 126.9780,
    timezone: "Asia/Seoul"
}
```

예를 들어 부산으로 바꾸고 싶다면
`locationName`, `latitude`, `longitude`를 부산 값으로 바꾸면 됩니다.

네, 필요합니다.

- 날씨 예보 조회

## 14. 실행 후 정상 동작 체크

현황판을 열었을 때 아래가 보이면 정상입니다.

- 현재 시간과 날짜가 보임
- 오늘 / 내일 날씨가 보임
- 오늘 학교 시간표가 보임
- `cram/cram.md`에 적은 오늘 학원 일정이 보임
- `StudySet 01`, `StudySet 02`에 카드 2개씩 번갈아 보임
- `local_secrets.env`를 넣었다면 매일 다른 2개가 의미 있게 선택됨
- 할일 / 숙제 / 약속 중 하나 이상이 보임

만약 학원 일정만 비어 있으면 가장 먼저 `cram/cram.md` 형식을 확인해 주세요.

## 15. 문제가 생겼을 때 가장 먼저 확인할 것

### 1. 화면은 열리는데 할일이 안 보일 때

- 인터넷 연결 확인
- `dashboard-config.js`의 Firebase 설정 확인
- 실제로 Firebase에 데이터가 저장되어 있는지 확인

### 2. 학원 일정이 안 보일 때

- `cram/cram.md` 파일이 있는지 확인
- 요일 제목이 `## 월요일` 형식인지 확인
- 일정 줄이 `- 16:30-18:00 | 수학` 형식인지 확인

### 3. 날씨가 안 보일 때

- 인터넷 연결 확인
- 잠시 기다렸다가 새로고침
- `dashboard-config.js`의 위도/경도 확인

### 4. 암기 카드가 안 보일 때

- `memory` 폴더 안에 md 파일이 있는지 확인
- `@interval:` 형식이 맞는지 확인
- `- 단어 | 뜻` 형식으로 적었는지 확인

### 5. Gemini 선택이 안 되는 것 같을 때

- `local_secrets.env` 파일이 있는지 확인
- 파일 안에 `GEMINI_API_KEY=...` 형식으로 입력했는지 확인
- `GEMINI_STUDYSET_SETUP.md`를 확인

### 6. `start_dashboard.bat` 실행이 안 될 때

- Python 설치 여부 확인
- 보안 프로그램이 `.bat` 실행을 막는지 확인
- 급하면 `index.html`을 직접 열어서 화면이 나오는지 먼저 확인

## 16. 앞으로 기능을 더 넣고 싶을 때

레이아웃을 유지하면서 기능을 추가하려면
`LAYOUT_EXTENSION_PROMPT.md` 파일의 문구를 기준으로 요청하면 됩니다.

그렇게 하면 지금처럼

- 정보량 많은 다크 테마
- 100vh 대시보드 구조
- 페이지 전체 스크롤 없는 구조
- 큰 빈 박스 없는 compact 레이아웃

을 유지하기 쉬워집니다.

## 17. 마지막 한 줄 요약

앞으로 가장 자주 만질 파일은 보통 2개입니다.

- 학교 시간표를 바꿀 때: `dashboard-config.js`
- 학원 일정을 바꿀 때: `cram/cram.md`
- 암기 카드를 바꿀 때: `memory` 폴더 안의 md 파일들
- Gemini 설정을 할 때: `local_secrets.env`, `GEMINI_STUDYSET_SETUP.md`

실행은 보통 `start_dashboard.bat`만 더블클릭하면 됩니다.
